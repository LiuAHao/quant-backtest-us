## 问题 2：宽基指数成分股缺失

### 问题描述

`index_member` 表中没有沪深300、中证500、中证1000、上证50的成分股数据。原因是 Tushare 代理不支持 `index_member(index_code=...)` 查询。

### Baostock 测试结果 ✅ 推荐

| 指数 | API | 当前成分股 | 历史成分股 | 耗时 |
|---|---|---|---|---|
| 沪深300 | `bs.query_hs300_stocks()` | ✅ 300 只 | ✅ 支持按日期查询 | 0.11s |
| 中证500 | `bs.query_zz500_stocks()` | ✅ 500 只 | ✅ 支持按日期查询 | 0.12s |
| 上证50 | `bs.query_sz50_stocks()` | ✅ 50 只 | ✅ 支持按日期查询 | 0.09s |
| 中证1000 | - | ❌ 不支持 | ❌ | - |

**历史成分股验证：**
- `bs.query_hs300_stocks(date='2024-06-30')` → 300 只，0.11s ✅
- `bs.query_hs300_stocks(date='2020-06-30')` → 300 只，0.10s ✅

**数据格式：**
```
updateDate      code code_name
2024-06-24 sh.600000      浦发银行
```
- `code` 格式为 `sh.600000`，需转换为 `600000.SH`

**批量下载评估：**
- 每次查询 0.1s，按月查询 12 年（144 个月）× 3 个指数 = 432 次查询
- 预计耗时：432 × 0.15s ≈ 65 秒（含网络延迟）
- 数据量：约 432 × 300 ≈ 130,000 条
- **可靠性：Baostock 稳定，无超时风险**

### AkShare 测试结果

| 指数 | API | 当前成分股 | 历史成分股 | 耗时 |
|---|---|---|---|---|
| 沪深300 | `ak.index_stock_cons('000300')` | ✅ 300 只 | ❌ 不支持 | 1.50s |
| 中证1000 | `ak.index_stock_cons('000852')` | ✅ 1000 只 | ❌ 不支持 | 4.48s |
| 上证50 | `ak.index_stock_cons('000016')` | ✅ 50 只 | ❌ 不支持 | ~1.5s |

**历史成分股：**
- `ak.index_stock_cons_csindex(symbol='000300')` 只返回当前成分股，不支持 `start_date` 参数

**缺点：**
- 不支持历史成分股查询（只能获取当前）
- 单次查询耗时较长（1.5-4.5s）
- 用户反馈大批量调用容易超时

### 推荐方案：Baostock

**优势：**
- 支持按日期查询历史成分股
- 速度快（0.1s/次）
- 稳定可靠，无超时风险

**劣势：**
- 不支持中证1000（只有 CSI300、CSI500、SSE50）
- 代码格式需要转换（`sh.600000` → `600000.SH`）

**数据量估算：**

| 指数 | 查询次数 | 每次条数 | 总条数 |
|---|---|---|---|
| 沪深300 | 144 月 | 300 | 43,200 |
| 中证500 | 144 月 | 500 | 72,000 |
| 上证50 | 144 月 | 50 | 7,200 |
| **合计** | 432 次 | - | **122,400** |

**预计耗时：** 约 1-2 分钟（Baostock 速度很快）

**中证1000 的替代方案：**
- 中证1000 成立于 2014 年 10 月，Baostock 不支持
- 可用 AkShare `ak.index_stock_cons('000852')` 获取当前成分股（1000 只）
- 历史成分股暂无免费数据源，需用市值近似

### 结论

✅ **推荐使用 Baostock**，数据量小、速度快、稳定可靠。中证1000 需要单独处理。

## 问题 3：财报为修正后数据

### 问题描述

Tushare 返回的是最新修正版财报，不是最初发布版。

### Baostock 测试结果

Baostock 的财务接口提供 `pubDate`（公告日期）和 `statDate`（报告期）：

```python
bs.query_profit_data(code='sh.600000', year=2024, quarter=1)
# code    pubDate   statDate   roeAvg  netProfit  epsTTM  ...
# sh.600000 2024-04-30 2024-03-31 0.0237 17660000000  1.3045
```

**关键字段：**
- `pubDate`: 公告日期（可用于 ann_date 过滤）
- `statDate`: 报告期截止日

**但是：** Baostock 同样返回修正后的数据，不是最初发布版。`pubDate` 只是最近一次公告的日期，不代表数据是原始版本。

**批量查询速度：** 0.12s/只，5 只股票 0.62s

### AkShare 测试结果

AkShare 的财务接口（同花顺数据源）返回详细财报：

```python
ak.stock_financial_benefit_ths(symbol='600000', indicator='报告期')
# 30 条，涵盖 2025-1996 年
```

**但是：** 同样返回修正后的数据，没有"原始版"标记。

### 结论

❌ **两个数据源都无法提供原始版财报**。这是数据源的根本限制，不是 API 参数的问题。

**缓解措施：**
- 使用 `ann_date <= trade_date` 过滤（已实现）
- 虽然是修正后数据，但至少不会用到"未来"的数据
- 对于大多数策略，修正幅度通常较小，影响有限
