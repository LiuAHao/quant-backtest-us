- 报告：`strategies/reports/strategy_20250101_20260429_210234.html`

## 本组结论

### 结论1：40日反转更适合周调仓，不适合月调仓
周调仓版本在收益和夏普上都明显优于月调仓版本，而回撤几乎没有变差：
- 周调仓：28.68% / -19.89% / 0.91
- 月调仓：12.60% / -20.13% / 0.50

说明40日反转虽然属于中等周期修复逻辑，但信号更新速度仍然不能太慢。周频能够较好接住反转修复，月频则明显钝化。

### 结论2：月调仓上叠加保留机制，本组没有带来正贡献
月调仓 + 保留机制相比纯月调仓：
- 收益从 12.60% 降至 10.72%
- 最大回撤从 -20.13% 恶化到 -24.53%
- 夏普从 0.50 降至 0.44

这说明对40日反转来说，月调仓本身已经偏慢，再增加“当前持仓若仍在前10则保留”的组合缓冲，会进一步延缓组合更新，使部分修复完成后的仓位留得过久。

### 结论3：低PB更像风险过滤器，而不是收益增强器
在月调仓 + 保留机制的基础上增加低PB后：
- 收益：10.72% → 10.91%
- 最大回撤：-24.53% → -15.27%
- 夏普：0.44 → 0.46

收益提升很有限，但回撤改善明显，说明低PB在这条线上更像风险约束条件，主要作用是减少组合中的高风险/高估值弱质反转样本，而不是显著放大收益。

## 当前阶段判断
- 第一组最优先版本：`mainboard_reversal40_annual_profit_low_pe_weekly_v2.py`
- 若偏向收益/效率平衡，优先继续沿“40日反转 + 年报盈利 + 低PE + 周调仓”主线深化。
- 若偏向稳健控制回撤，可保留“低PB + 月调仓 + 保留机制”作为风险更收敛的备选分支。

## 后续建议
下一步按对称结构继续跑第二组（20日反转对应的周调仓 / 月调仓 / 月调仓保留 / 月调仓保留+低PB），比较20日与40日反转在调仓频率和过滤器响应上的差异。

## 本组结论

### 结论1：20日反转这条线，月调仓明显优于周调仓
- 周调仓：20.80% / -23.70% / 0.68
- 月调仓：32.72% / -21.20% / 1.01

说明20日反转在当前样本里并不需要更高频率更新。周频反而容易引入更多短期噪音，而月频在年报盈利和低PE过滤后，组合表现更稳、更顺。

### 结论2：保留机制在第二组里基本没有产生实际影响
月调仓和月调仓+保留机制三项指标完全一致：
- 收益：32.72%
- 最大回撤：-21.20%
- 夏普：1.01

这说明在该样本区间内，保留机制没有实质改变组合路径，至少不是当前第二组的关键变量。

### 结论3：低PB在20日反转上是明显有效增强，不只是控回撤
在月调仓+保留机制基础上增加低PB后：
- 收益：32.72% → 41.99%
- 最大回撤：-21.20% → -19.13%
- 夏普：1.01 → 1.23

这说明低PB在20日反转线上不是单纯的风险过滤器，而是显著改善收益/回撤比的增强条件。

## 两组合并判断
- 40日反转更依赖调仓频率匹配，周频优于月频。
- 20日反转更依赖过滤器清理样本，月频优于周频。
- 40日线中低PB主要改善回撤；20日线中低PB同时提升收益和夏普。

## 当前阶段判断
- 40日线当前最优先版本：`mainboard_reversal40_annual_profit_low_pe_weekly_v2.py`
- 20日线当前最优先版本：`mainboard_reversal20_annual_profit_low_pe_pb_monthly_hold_v1.py`

从本轮结果看，20日线最优版本（41.99% / -19.13% / 1.23）优于40日线最优版本（28.68% / -19.89% / 0.91）。

## 对“20日线是否每日会更好”的当前判断
从这轮对照结果本身看，不能直接推出“20日线每日会更好”，反而当前证据更支持：
- 周频不如月频；
- 20日线的改善更可能来自过滤器组合（年报盈利 + 低PE + 低PB）而不是单纯提高换手频率。

因此，若要验证“每日是否更好”，应单独做一轮同底座归因测试：
- 保持股票池、过滤器、持仓数不变；
- 仅把月调仓改成日调仓；
- 再与周频、月频直接对照。

在没有这组同底座对照前，不能先下“20日线日频更优”的结论。

### 2. 40日反转 + 年报盈利 + 周调仓
- 策略文件：`backend/storage/strategies/mainboard_reversal40_annual_profit_weekly_v1.py`
- 报告：`strategies/reports/strategy_20250101_20260429_234017.html`

## 本组结论

- 20日：100.45% / -24.08% / 1.90
- 40日：70.33% / -20.02% / 1.59

说明在当前样本区间内，如果不叠加估值过滤器，只保留年报盈利这一层样本质量约束，那么20日反转的收益弹性和收益风险比都优于40日反转。

### 结论2：40日反转不是无效，而是更偏稳一点的中周期修复
40日版本仍然跑出了 70.33% / 1.59 夏普，说明这条线本身也成立；只是相较20日，它：
- 回撤略小；
- 收益弹性更弱；
- 更像中周期修复，而不是更短周期的强弹性超跌反转。

### 结论3：20日纯年报主线的强势，并不完全依赖日频硬切
此前的 `mainboard_reversal20_annual_profit_v1.py`（日频硬切）结果是：
- 110.70% / -20.95% / 2.05

- 100.45% / -24.08% / 1.90

这说明 20日反转 + 年报盈利 这条主线本身就很强，不是只有在日频高换手执行下才成立。

## 对当前研究框架的影响
这组结果把一个很重要的问题拆清了：

1. **如果只比较反转窗口本身**（不叠加估值过滤器），当前样本里 **20日 > 40日**。
2. **如果叠加 low PE / low PB 等估值过滤器**，20日与40日的相对强弱可能会被改写。
3. 因此，后续讨论“20日强还是40日强”时，必须先说清楚：
   - 是在纯年报过滤底座下比较；
   - 还是在估值增强后的优化底座下比较。

## 当前阶段口径更新
可以把当前研究结论更新为两层：

- **纯年报过滤主线**：20日反转强于40日反转；
- **加入估值过滤后的优化主线**：40日与20日对 `low PE` / `low PB` 的响应不同，不能直接沿用纯年报结论。

- 收益：100.45%

## 结论整理

### 结论1：这条策略不是长期稳定有效，而是明显带有阶段性
短区间表现非常强，但长区间直接转负且回撤极深，说明“20日反转 + 年报盈利 + 周调仓”并不是一个已经验证过的长期稳健策略。

### 结论2：它更适合牛市/风险偏好回升阶段，不适合熊市环境
结合短区间与长区间表现，可以把当前对该策略的市场适用性理解为：
- 在牛市、修复市、风险偏好显著回升阶段，20日超跌修复容易快速兑现，因此策略能够表现出较强弹性；
- 在熊市或持续下行环境中，超跌本身并不意味着马上修复，反而容易不断“越跌越选”，导致策略难以生效，回撤显著放大。

### 结论3：当前应把它定义为“阶段性有效策略”，而不是“长期通用策略”
因此后续在研究或实盘观察中，更适合把这条线作为：
- **牛市/反弹市/风险偏好修复阶段的进攻型策略候选**；
而不是：
- 跨完整牛熊周期都能稳定持有的长期底仓型策略。

## 对研究框架的更新
这条补充会把此前“20日纯年报主线很强”的结论再约束一层：
- 在近端样本和特定市场环境里，它确实很强；
- 但从 2020-01-01 拉长到 2026-04-29 后，它并不具备稳定穿越能力；
- 因此后续讨论这条策略时，应始终附带一句：**它更适合牛市/修复市，不适合熊市。**
